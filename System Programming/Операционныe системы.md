Операционныe системы предоставляют приложениям простой и унифицированный интерфейс для работы с аппаратурой, скрывая сложность низкоуровневых операций.
* Пример: вместо того чтобы писать код для управления жестким диском, программа использует функции ОС для чтения и записи файлов.
### 3. Обеспечение многозадачности:
* ОС позволяет нескольким программам выполняться одновременно, даже если компьютер имеет только один процессор.
* Пример: вы можете слушать музыку, печатать документ и скачивать файл одновременно.
* Управление памятью:
* ОС выделяет память для программ и следит за тем, чтобы одна программа не могла повредить память другой.
* Пример: ОС предотвращает доступ программы к памяти, выделенной для другой программы.
### 4. Обеспечение безопасности:
* ОС защищает данные и ресурсы от несанкционированного доступа.
Пример: ОС требует ввода пароля для доступа к учетной записи пользователя.
### 5. Управление файловой системой:
* ОС организует хранение данных на диске, предоставляя удобный интерфейс для работы с файлами и каталогами.
* Пример: вы можете создавать, удалять и перемещать файлы с помощью проводника.

## Основные концепции операционных систем

### 2.1. Процессы и потоки
* Процесс — это программа, которая выполняется в отдельном адресном пространстве. Каждый процесс имеет собственные ресурсы (память, файловые дескрипторы и т.д.).
* Поток (thread) — это часть процесса, которая может выполняться параллельно с другими потоками. Потоки разделяют ресурсы процесса.
Пример:
* Браузер может быть процессом, а каждая вкладка — отдельным потоком.
### 2.2. Управление памятью
* ОС управляет оперативной памятью, выделяя её процессам и следя за тем, чтобы процессы не мешали друг другу.
* Основные механизмы:
	* Виртуальная память: каждый процесс "видит" свою собственную память, изолированную от других процессов.
	* Свопинг (swapping): если памяти не хватает, ОС перемещает часть данных на диск (в файл подкачки).
### 2.3. Файловая система
* Файловая система организует хранение данных на диске. Она предоставляет структуру каталогов и файлов, а также механизмы для доступа к ним.
* Примеры файловых систем: NTFS (Windows), ext4 (Linux), APFS (macOS).
### 2.4. Управление устройствами
* ОС управляет устройствами ввода-вывода (клавиатура, мышь, принтер, диски и т.д.) через драйверы устройств.
* Драйвер — это программа, которая "переводит" команды ОС на язык, понятный устройству.
### 2.5. Многозадачность
* ОС позволяет нескольким программам выполняться "одновременно". Это достигается за счет:
	* Вытесняющей многозадачности: ОС решает, какой процесс будет выполняться в данный момент.
	* Кооперативной многозадачности: процессы сами передают управление друг другу (редко используется в современных ОС).
### 2.6. Безопасность
* ОС обеспечивает защиту данных и ресурсов с помощью:
	* Учетных записей пользователей: каждый пользователь имеет свои права доступа.
	* Разграничения прав: например, только администратор может устанавливать программы.

## Типы операционных систем

Операционные системы можно классифицировать по различным критериям:
### 1. По количеству пользователей:
* Однопользовательские (например, Windows для домашнего использования).
* Многопользовательские (например, серверные ОС, такие как Linux).
### 2. По количеству задач:
* Однозадачные (например, MS-DOS).
* Многозадачные (например, Windows, macOS, Linux).
### 3. По типу использования:
* Настольные ОС (Windows, macOS, Linux для домашних компьютеров).
* Серверные ОС (Windows Server, Linux для серверов).
* Встроенные ОС (например, Android для смартфонов, FreeRTOS для микроконтроллеров).

## Архитектура операционной системы

Операционная система состоит из нескольких слоев:
### 1. Ядро (Kernel):
* Это центральная часть ОС, которая управляет ресурсами и обеспечивает взаимодействие между аппаратурой и программами.
* Функции ядра:
	* Управление процессами.
	* Управление памятью.
	* Управление устройствами.
	* Обработка системных вызовов.
### 3. Системные библиотеки:
* Это набор функций, которые программы могут использовать для взаимодействия с ядром.
* Пример: библиотека `glibc` в Linux.
### 4. Интерфейс пользователя:
* Это слой, через который пользователь взаимодействует с ОС.
* Примеры:
	* Графический интерфейс (GUI): рабочий стол, окна, кнопки.
	* Командная строка (CLI): текстовый интерфейс для ввода команд.
### 5. Системные утилиты:
* Это программы, которые помогают управлять системой (например, диспетчер задач, утилиты для работы с дисками).

## Примеры операционных систем
### 1. Windows:
* Разработана Microsoft.
* Широко используется на настольных компьютерах и ноутбуках.
* Поддерживает множество приложений и игр.
### 2. macOS:
* Разработана Apple.
* Используется на компьютерах Mac.
* Известна своей стабильностью и удобством.
### 3. Linux:
* Это семейство открытых ОС на основе ядра Linux.
* Широко используется на серверах, в embedded-системах и как альтернатива Windows на настольных компьютерах.
* Примеры дистрибутивов: Ubuntu, Fedora, Debian.
### 4. Android: 
* Мобильная ОС на основе ядра Linux.
* Разработана Google.
* Используется на смартфонах и планшетах.

## Как работает операционная система?
### 1. Загрузка: 
* При включении компьютера BIOS/UEFI загружает ядро ОС в память. 
* Ядро инициализирует систему и запускает начальные процессы.
### 2. Работа с пользователем: 
* ОС предоставляет интерфейс (графический или командный) для взаимодействия с пользователем.
### 3. Выполнение программ: 
* ОС загружает программы в память и управляет их выполнением.
### 4. Обработка прерываний: 
* ОС реагирует на события (например, нажатие клавиши или завершение операции ввода-вывода) с помощью прерываний.
### 5. Завершение работы: 
* ОС корректно завершает все процессы и выключает компьютер.>)
