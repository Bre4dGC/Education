#### **1. Основные компоненты компьютера**  
**Центральный процессор (CPU)**  
- **Главная задача**: Выполнение программ (последовательное чтение и обработка команд).  
- **Аналогия**: Как человек читает книгу слово за словом, CPU читает байты из памяти.  

**Память**  
- **ROM (Read-Only Memory)**:  
  - Хранит прошивку (например, BIOS).  
  - Данные сохраняются после выключения питания.  
- **RAM (Random Access Memory)**:  
  - Временное хранение данных во время работы.  
  - Стирается при отключении питания.  

**Пример**:  
- **BIOS** — программа в ROM, запускается при включении ПК.  
- **Оперативная память** (DDR4/DDR5) — это RAM.  
---
#### **2. Как CPU общается с памятью**  
**Адресное пространство**  
- Память организована в виде массива байтов (каждый байт имеет уникальный адрес, начиная с 0).  
- **Пример**: Адрес 299 соответствует 300-й ячейке (нумерация с 0).  

**Шины передачи данных**  
1. **Адресная шина (Address Bus)**:  
   - Передаёт номер ячейки памяти, к которой обращается CPU.  
   - **Разрядность**: Определяет максимальный объём адресуемой памяти.  
     - 16-битная шина → 64 КБ (2^16 = 65,536 ячеек).  
2. **Шина данных (Data Bus)**:  
   - Передаёт сами данные (например, 8 бит = 1 байт).  
   - **Разрядность**: 8-бит, 16-бит, 32-бит, 64-бит (влияет на производительность).  
3. **Шина управления (Control Bus)**:  
   - Сигналы `RD` (чтение) и `WR` (запись).  
   - **Активный уровень**: Низкий (0V = активация).  
---
#### **3. Пример работы CPU с памятью**  
**Шаг 1: Чтение из ROM**  
- CPU выставляет адрес (например, `44206` в двоичном виде) на адресную шину.  
- Активирует сигнал `RD`.  
- ROM возвращает значение байта (например, `112` → команда для CPU).  

**Шаг 2: Запись в RAM**  
- CPU отправляет адрес (`62088`) и данные (`205`) через шины.  
- Активирует сигнал `WR`.  
- RAM сохраняет значение в указанную ячейку.  
---
#### **4. Декодирование адресов**  
**Проблема**: Как CPU различает ROM и RAM, если они подключены параллельно?  
**Решение**:  
- **Декодер адресов**: Логическая схема, активирующая нужное устройство.  
  - **Пример**:  
    - ROM: Адреса `0x0000–0x7FFF` (младший бит `A15 = 0`).  
    - RAM: Адреса `0x8000–0xFFFF` (старший бит `A15 = 1`).  
- **Сигнал `CS` (Chip Select)**: Активирует конкретную микросхему памяти.  
---
#### **5. Работа с портами ввода-вывода**  
**Пример: Мигание светодиодом**  
- Добавляем **порт вывода** (например, адрес `0xFF`).  
- CPU записывает `1` или `0` в порт → светодиод включается/выключается.  
- **Декодирование**:  
  - Адрес `0xFF` = `11111111` в двоичном виде.  
  - Сигналы `WR` и `IO` (ввод-вывод) должны быть активны.  

**Аналогия**:  
- Так работали старые ISA-слоты (например, Sound Blaster).  
- Современные PCIe использует отдельные линии для каждого устройства.  
---
#### **6. Шестнадцатеричная система (Hex)**  
- Упрощает запись двоичных чисел.  
- **Пример**:  
  - `11011010` (бинарно) = `DA` (hex).  
  - В программировании: `0xDA` (C), `DAh` (ассемблер).  
---
### **Заключение**  
- CPU выполняет программы, читая команды из памяти через **адресную**, **данную** и **управляющую** шины.  
- **Декодирование адресов** позволяет разделять память и устройства ввода-вывода.  
- **8-битные, 16-битные, 32-битные системы** — это про разрядность шины данных.  
- **Современные компьютеры** используют те же принципы, но с более сложной организацией (например, многоуровневый кэш).  

**Совет**: Чтобы глубже понять архитектуру, изучите:  
1. Работу **Z80** или **6502** (классические 8-битные CPU).  
2. Программирование **Arduino** (практика с портами ввода-вывода).
