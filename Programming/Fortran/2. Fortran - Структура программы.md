Каждая Fortran-программа должна иметь главную подпрограмму. Из главной подпрограммы вызываются другие подпрограммы, составляющие программу. Синтаксис главной программы следующий:
```fortran
program [program-name]
[specification statements]
[executable statements]
[contains]
[subroutines]
end [program [program-name]]
```
Из синтаксиса видно, что единственным идентификатором, который должен быть включен в определение основной программы, является end.

Синтаксис подпрограмм и функций определяется аналогичным образом, но идентификатор программы заменяется на подпрограмму или функцию. Правильным способом организации подпрограмм является их размещение в отдельных файлах или в модулях (об этом будет рассказано в следующих разделах). Подпрограммы также можно поместить в раздел contains основной программы, что является предпочтительным методом, если все подпрограммы размещаются в одном исходном файле. В приведенном ниже коде показан простой пример основной программы с подпрограммой на языке Фортран.
```fortran
program sample1
	
    integer, parameter :: dp=selected_real_kind(15,300)
    real(kind=dp) :: x, y
    real(kind=dp) :: k(20,20)
	
    x = 6.0_dp
    y = 0.25_dp
	
    write(*,*) x
    write(*,*) y
    write(*,*) dp
	
    call myproc(k)
	
    write(*,*) k(1,1)
	
contains
	
subroutine myproc(k)
	
    integer, parameter :: dp=selected_real_kind(15,300)
    real(kind=dp) :: k(20,20)
	
    k=0.0_dp
    k(1,1) = 42.0_dp
	
    return
	
end subroutine myproc
	
end program sample1
```
Исходный код программы может содержать прописные и строчные буквы, цифры и специальные символы. Однако следует отметить, что Фортран не различает прописные и строчные буквы. Исходный текст программы записывается, начиная с первой позиции, с одним оператором в каждой строке. Если строка завершается символом &, это означает, что утверждение продолжается на следующей строке. Весь текст, расположенный после символа !, является комментарием и не влияет на работу программы. Даже если комментарии не несут никакой функции в программе, они важны для читаемости исходного кода. Это особенно важно для будущей модификации программы. Помимо описанной выше формы исходного кода существует также возможность написания кода в фиксированной форме, как в Fortran 77 и более ранних версиях. В предыдущей версии стандарта Fortran это была единственная доступная форма исходного кода.
