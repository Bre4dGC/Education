Числовые выражения в Фортране состоят из операндов встроенных типов данных **integer**, fkeyw{real} или **complex**.

Если операнды состоят только из целых чисел, важно отметить, что целочисленные деления округляются до 0. Это иллюстрирует следующий пример:
```fortran
program expr1
    print*, 6 / 3
    print*, 8 / 3   ! 2.6666... округлим до 2
    print*, -8 / 3  ! -2.666... округлим до 2
end program expr1
```

В результате выполнения программы вы получите следующий результат:
```
2
2
-2
```

При работе с целочисленными выражениями нужно быть осторожным и с другими моментами. Рассмотрим следующий пример:
```fortran
program expr2
    print*, 2 ** 3
    print*, 2 ** (-3)
end program expr2
```

При запуске будет выдан следующий результат:
```
8
0
```
Причина появления 0 во втором выражении заключается в том, что **2**(-3)** равно 1/2**3, которое в целочисленном выражении будет усечено до 0.

При смешивании типов данных в выражениях более слабые типы данных будут преобразованы в более сильный тип. Результат выражения будет иметь более сильный тип. **real** сильнее, чем **integer**. Рассмотрим следующий код:
```fortran
real(dp) :: a
integer :: i
real(dp) :: b

b = a * i
```
Здесь ссылка на переменную i будет преобразована в **real(dp)** при вычислении выражения.
